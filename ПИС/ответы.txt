1.GO: определение, свойства системы программирования.
_Go (Golang) – система программирования, 2009 (Google)Роберт Гриземер, Роб Пайк и Кен Томпсоном, для системного и серверного программирования, особенности: простой, статическая типизация, встроенная поддержка параллелизма, высокая производительность, мощная стандартная библиотека.
_GO: удобный для web-сервера:
­компилируется в нативный код (быстрый);
­простая и эффективная (встроенная, goroutine) многопоточность (собственный M:N –планировщик, M >> N);
­встроенные HTTP и RPC – пакеты; 
­легкий деплоймент (один бинарный файл, без внешних зависимостей);
­стандартная библиотека статическая (линкуется в итоговый бинарник);
­Runtime GO встроен в бинарник, сгенерированный бинарный файл не требует никаких зависимостей; 
­один компилятор генерирует нативный код (исполняемые файлы) для различных платформ (ОС, архитектура);
­можно использовать dll/so, код GO – не меняется;
­одна утилита CLI (Command Language Interface) go.       
Стандартная библиотека в виде файликов с расширением .go.
_Go (Golang) – это компилируемый язык программирования общего назначения, разработанный компанией Google в 2007 году и официально представленный в 2009 году.
_Зачем создавался Go?
В Google столкнулись с проблемой:
Крупные проекты на C++ становились:
слишком большими;
долго компилировались;
сложными для поддержки;
сложными для параллельного программирования.
Требовался язык, который бы сочетал:
Производительность C/C++
Простоту Python
Надежность Java
Удобную работу с многопоточностью
Так появился Go.
_Go – это статически типизированный компилируемый язык программирования общего назначения с автоматическим управлением памятью, встроенной поддержкой конкурентного программирования и высокой скоростью выполнения программ.
Под системой программирования понимают не только язык.
Она включает:
Компилятор
Средства сборки
Средства управления пакетами
Стандартную библиотеку
Средства тестирования
Средства форматирования кода
Средства документирования
_Основные свойства Go
1) Компилируемый язык. Go компилируется непосредственно в машинный код (Высокая скорость выполнения. Нет необходимости устанавливать виртуальную машину.)
2) Статическая типизация. Тип переменной определяется во время компиляции (меньше ошибок; быстрее выполнение; лучше оптимизация.)
3) Строгая типизация. Go не делает опасных автоматических преобразований.
4) Автоматическое управление памятью. В Go есть Garbage Collector. Он автоматически удаляет неиспользуемые объекты.
5) Высокая производительность. Go компилируется в машинный код. По скорости обычно близок к: C, С++ и быстрее Python, JavaScript, PHP
6) Простота языка. Создатели сознательно сделали язык небольшим.
В Go:
мало ключевых слов;
нет наследования классов;
нет перегрузки методов;
нет перегрузки операторов.
Всего около 25 ключевых слов.
7) Встроенная конкурентность. Одно из главных достоинств Go. Goroutine – легковесный поток выполнения.
8) Каналы (Channels). Используются для обмена данными между goroutine. Идея Go: Don't communicate by sharing memory; share memory by communicating. То есть: Не делитесь памятью между потоками, а передавайте данные через каналы.
9) Кроссплатформенность. Из одной ОС можно собрать программу под другую. Из Windows собрать Linux-приложение:
set GOOS=linux
set GOARCH=amd64
go build
Поддерживаются:
Windows
Linux
macOS
FreeBSD
ARM
ARM64
и другие платформы.
10) Богатая стандартная библиотека. Поэтому многие задачи решаются без сторонних библиотек.
11) Встроенные инструменты разработки
12) Модульная организация программ. Код разбивается на пакеты.
_Go (Golang) – это статически типизированный компилируемый язык программирования общего назначения, разработанный Google. Основными свойствами системы программирования Go являются высокая производительность, автоматическое управление памятью, строгая и статическая типизация, встроенная поддержка конкурентного программирования через goroutine и каналы, кроссплатформенность, развитая стандартная библиотека и наличие встроенных средств сборки, тестирования, форматирования и управления зависимостями. Go широко применяется для разработки серверных приложений, сетевых сервисов, микросервисов и облачных систем.

2.GO: CLI: основные команды.
_CLI (Command Line Interface) – это интерфейс командной строки, позволяющий управлять системой программирования Go через команды в терминале.
После установки Go появляется исполняемый файл: go
Это главный инструмент разработчика.
Через него выполняются:
компиляция;
запуск;
тестирование;
создание модулей;
загрузка зависимостей;
генерация документации;
форматирование кода;
анализ кода.
Можно сказать:
Команда go – это одновременно компилятор, менеджер пакетов, система сборки и средство тестирования.
Любая команда выглядит так: go <команда> [аргументы]
_Самые важные команды
1) go version. Показывает установленную версию Go. 
Чтобы проверить:
установлен ли Go;
какая версия используется;
под какую платформу работает.
2) go env. Показывает настройки окружения Go.
GOOS – целевая операционная система.
GOARCH – архитектура процессора.
GOROOT – где установлен Go.
GOPATH – рабочее пространство пользователя.
3) go help. Показывает справку. Получаем подробное описание команд.
4) go run. Самая популярная команда у начинающих. Компилирует программу во временный файл и сразу запускает её.
Что происходит внутри?
main.go
    ↓
Компиляция
    ↓
Временный exe
    ↓
Запуск
    ↓
Удаление exe
Когда использовать? Во время разработки.
Когда не использовать? Для выпуска готового приложения.
5) go build. Создает исполняемый файл.
Исходный код
      ↓
Компиляция
      ↓
Линковка
      ↓
EXE
6) go install. Похожа на build. Дополнительно помещает программу в каталог бинарников. После программу можно запускать из любого места. 
Используют для установки:
собственных утилит;
сторонних инструментов.
7) go test. Запускает тесты. Тестирование встроено в язык. Не нужны дополнительные инструменты.
8) go fmt. Автоматически форматирует код. Во многих командах и компаниях считается: Если код не проходит go fmt – он не готов к коммиту.
9) go vet. Статический анализатор. Ищет подозрительный код.
Например:
fmt.Printf("%d", "hello")
Компилятор может пропустить.
Но go vet скажет:
Printf format %d has arg string
Что ищет?
подозрительные конструкции;
ошибки форматирования;
неправильное использование функций;
потенциальные баги.
10) go clean. Удаляет результаты сборки. Полезно при проблемах со сборкой. Удаляет:
exe
объектные файлы
кэш сборки
11) go doc. Показывает документацию.
Пример: go doc fmt.Println
Получаем: func Println(a ...any) (n int, err error)
Это встроенный аналог справочника.
12) go mod. Очень важная команда. Система управления модулями.
Модуль – это проект Go.
Зачем нужен go.mod?
В нем хранится:
имя проекта;
версия Go;
зависимости.
Основные подкоманды go mod
А) go mod init. Создает модуль.
go mod init myproject
Б) go mod tidy. Добавляет недостающие зависимости и удаляет ненужные.
В) go mod download. Скачивает зависимости.
Г) go mod verify. Проверяет целостность зависимостей.
Д) go mod graph. Показывает дерево зависимостей.
CLI Go представляет собой утилиту go, через которую выполняется управление проектами, сборка, запуск и тестирование программ. Основными командами являются: go run – запуск программы, go build – создание исполняемого файла, go install – установка программы, go test – запуск тестов, go fmt – форматирование кода, go vet – статический анализ, go mod – управление модулями и зависимостями, go env – просмотр настроек окружения, go doc – получение документации и go clean – очистка результатов сборки. Система CLI является одной из ключевых особенностей Go, так как объединяет компилятор, систему сборки и менеджер пакетов в одном инструменте.

3. GO: структура проекта (директории, конфигурационные файлы)
Структура проекта – это организация файлов и каталогов приложения.
Она определяет:
где находится исходный код;
где находятся зависимости;
где хранятся конфигурации;
где располагаются тесты;
как проект собирается.
В Go нет жестко обязательной структуры проекта.
Однако со временем в сообществе появились рекомендации по организации крупных проектов.
Поэтому существуют:
1) Минимальная структура
2) Стандартная (рекомендуемая) структура
_Минимальная структура проекта

Самый простой проект:
hello/
│
├── go.mod
└── main.go
_Основные элементы структуры проекта
Практически в любом проекте будут:
project/
│
├── main.go
├── go.mod
├── go.sum
└── другие каталоги
_Файл main.go
Точка входа приложения. Здесь начинается выполнение программы.
_Конфигурационный файл go.mod
Самый важный файл проекта. Создается командой: go mod init project
Что хранится в go.mod?
1) Имя модуля. 
module github.com/user/project
По этому имени другие проекты смогут импортировать ваш код.
Например:
import "github.com/user/project/service"
2) Версия Go. 
go 1.24
Говорит: проект рассчитан на работу с Go 1.24
3) Зависимости
Например:
require github.com/gin-gonic/gin v1.10.0
Здесь указано:
пакет
версия пакета
_Конфигурационный файл go.sum
Хранит контрольные суммы зависимостей.
Чтобы проверить:
что библиотека не повреждена;
что библиотеку не подменили;
что все разработчики используют одинаковую версию.

4. GO: структура приложения (пакеты, точка входа, функции, …)
Приложение Go – это набор файлов .go, объединенных в пакеты и модули. Минимальное приложение выглядит так:
package main
import "fmt"
func main() {
	fmt.Println("Hello, Go!")
}
Именно в этом примере присутствуют почти все основные элементы структуры приложения.
_Из чего состоит приложение Go?
Основные элементы:
Приложение
│
├── Пакеты (package)
│
├── Импорт (import)
│
├── Переменные (var)
│
├── Константы (const)
│
├── Типы (type)
│
├── Структуры (struct)
│
├── Интерфейсы (interface)
│
├── Функции (func)
│
└── Точка входа main()
_Что такое пакет?
Пакет – это механизм объединения связанных файлов и кода. Можно сказать: Пакет – это аналог пространства имен (namespace) в других языках.
Без них весь код находился бы в одном огромном файле. Пакеты позволяют разделять программу на части.
Например:
project
│
├── models
├── service
├── repository
└── handlers
Каждая часть отвечает за свою задачу.
Правило Go
Все файлы одной папки должны принадлежать одному пакету. Иначе компилятор выдаст ошибку.
Специальный пакет main
Особенный пакет.
package main
Говорит компилятору: из этого пакета можно собрать исполняемую программу.
Если написать: package math – то получится библиотека.
Если написать: package main – то получится приложение.
_Точка входа программы
В любой исполняемой программе должна быть функция:
func main()
Она является точкой входа. Именно с неё начинается выполнение.
_Импорт пакетов
Программа редко состоит только из собственного кода. Чаще используются библиотеки. Для подключения применяется: import
Несколько импортов
import (
	"fmt"
	"time"
	"os"
)
 
_Функции
Основной строительный блок программы.
Объявление:
func имя() {
}
Несколько возвращаемых значений – уникальная особенность Go.
_Переменные
Объявление: var age int
Инициализация: var age int = 20
Короткая форма: age := 20
_Константы
Не изменяются после создания.
const PI = 3.14
_Типы
Можно создавать собственные типы.
type UserID int
_Структуры (Struct)
Главный способ хранения объектов.
type User struct {
	Name string
	Age  int
}
Создание:
u := User{
	Name: "Диана",
	Age: 20,
}
Доступ:
fmt.Println(u.Name)
_Методы
Функции, привязанные к структуре.
func (u User) SayHello() {
	fmt.Println("Hello", u.Name)
}
_Интерфейсы
Позволяют описывать поведение объектов.
type Speaker interface {
	Speak()
}
Любая структура с методом Speak() автоматически реализует интерфейс. Реализация происходит автоматически.
Приложение Go состоит из пакетов, функций, типов данных и точки входа. Основной единицей организации кода является пакет (package), который объединяет связанные файлы. Исполняемое приложение должно содержать пакет main и функцию main(), являющуюся точкой входа программы. Для подключения библиотек используется оператор import. Логика приложения реализуется через функции, которые могут принимать параметры и возвращать значения. Кроме функций, в Go используются структуры (struct), методы (method) и интерфейсы (interface), позволяющие строить сложные приложения с четкой организацией кода.

5.GO: фундаментальные типы, объявления, константы.
Тип данных определяет:
какие значения может хранить переменная;
сколько памяти выделяется;
какие операции можно выполнять над данными.
Компилятор использует типы для:
проверки ошибок;
выделения памяти;
оптимизации программы.
_Классификация типов в Go
Все типы можно разделить на две большие группы.
Типы Go
│
├── Фундаментальные (встроенные)
│
└── Составные
_Фундаментальные типы Go
К ним относятся:
Числовые
Логические
Строковые
_Числовые типы
Делятся на:
Целочисленные
Вещественные
Комплексные
_Целочисленные типы
1) int. Самый используемый тип.
var age int = 20
Размер зависит от архитектуры. На современных системах обычно: 64 бита. Диапазон примерно: -9 квинтиллионов до +9 квинтиллионов
2) int8. 8 бит.
var a int8 = 127
Диапазон: -128 ... 127
3) int16. 16 бит.
var a int16 = 1000
4) int32. 32 бита.
var a int32 = 100000
5) int64. 64 бита.
var a int64 = 1000000000
6) uint
var count uint = 100
7) uint8
То же самое что:
byte
var b byte = 255
Диапазон: 0 ... 255
8) uint16
var a uint16 = 65535
9) uint32
var a uint32 = 100000
10) uint64
var a uint64 = 9999999999
11) Особый тип uintptr
Используется для хранения адресов памяти.
var ptr uintptr
Применяется редко. 
Обычно в:
runtime;
unsafe;
системном программировании.
_Вещественные типы
Используются для дробных чисел.
1) float32. 32 бита.
var pi float32 = 3.14
Точность: примерно 6–7 знаков.
2) float64. Самый используемый тип.
var pi float64 = 3.1415926535
Точность: примерно 15–16 знаков.
_Комплексные числа
Редко используются, но встроены в язык.
1) complex64
var c complex64 = 1 + 2i
2) complex128
var c complex128 = 5 + 3i
Мнимая единица: i
Пример: c := 2 + 4i
_Логический тип
1) bool
Хранит: true или false
var isAdmin bool = true
_Строковый тип
1) string
Строка в Go представляет собой последовательность байтов UTF-8.
var name string = "Диана"
Можно писать: name := "Диана"
Строки неизменяемы.
Нельзя: name[0] = 'A'
_Руны (rune)
rune = int32
Используется для хранения Unicode-символов.
var r rune = 'Ж'
Почему нужен rune?
Потому что символ может занимать несколько байтов.
s := "Привет"
_byte
Очень популярный тип. На самом деле: byte = uint8
Используется для:
файлов;
сетевых пакетов;
бинарных данных.
data := []byte("Hello")
_Объявление переменных
1) Полное объявление
var age int
Получает значение по умолчанию: 0
2) С инициализацией
var age int = 20
3) Вывод типа
var age = 20
Компилятор сам определяет: int
4) Короткое объявление
Самое популярное.
age := 20
Компилятор автоматически выводит тип.
var – можно использовать везде.
:= – только внутри функций. Нельзя вне функции. Ошибка компиляции.
_Групповое объявление
Можно объявить несколько переменных.
var (
    age  int
    name string
)
Или: var a, b int
Или: x, y := 10, 20
_Значения по умолчанию (Zero Value)
 
_Константы
Константа — это значение, которое нельзя изменить после создания.
const PI = 3.14
_Iota

Специальный счетчик констант.
const (
    Red = iota
    Green
    Blue
)
Результат:
Red   = 0
Green = 1
Blue  = 2
Другой пример:
const (
    KB = 1 << (10 * iota)
    MB
    GB
)
Получаем:
KB = 1024
MB = 1048576
GB = 1073741824
В Go фундаментальные типы данных включают целочисленные (int, int8, uint64 и др.), вещественные (float32, float64), комплексные (complex64, complex128), логический тип (bool) и строковый тип (string). Для объявления переменных используется ключевое слово var или сокращенная форма :=. Все переменные имеют значение по умолчанию (zero value). 6. GO: указатели, работа с указателями.
Указатель (pointer) – это переменная, которая хранит не само значение, а адрес другой переменной в памяти.
var p *int
Сохраним адрес переменной: p = &age
_Зачем нужны указатели?
1) Изменение переменной из другой функции. Функция работает с оригинальной переменной, а не с ее копией.
2) Экономия памяти. Опять же, когда мы передаем, например, структуру по значению, то создается ее копия. А при передаче указателя мы передаем ту же самую структуру, ничего нового не создается. Размер адреса обычно: 8 байт. Поэтому большие структуры почти всегда передают через указатели.
_Адрес памяти
Каждая переменная имеет адрес. Получить адрес можно оператором: &
_Оператор *
Второй важнейший оператор. Используется для получения значения по адресу. То есть перейти по адресу и взять значение. Или еще по-другому разыменование указателя.
Можно также изменять значение с помощью указателя.
_Nil-указатели
Как и другие типы, указатели имеют значение по умолчанию.
var p *int
Такой указатель будет иметь значение nil, то есть он ни на что не указывает. При попытке разыменования пустого указателя мы получим panic, потому что адрес отсутствует.
_Создание объекта через new()
Go предоставляет встроенную функцию: new()
p := new(int)
Создается память под int. Тут p хранит адрес.
_Указатели на структуры
Самое частое применение указателей.
Доступ через указатель к полю: p.Name
Хотя формально: (*p).Name
Go автоматически делает разыменование.
_Методы с указателями
type User struct {
	Name string
}
Метод:
func (u *User) ChangeName(name string) {
	u.Name = name
}
Использование:
user := User{
	Name: "Анна",
}
user.ChangeName("Диана")
Результат: Диана
Почему используется *User?
Чтобы изменять исходную структуру.
Если написать func (u User) ChangeName(...), то будет работать копия объекта.
_В Go нет арифметики указателей
Если попытаться, то будет ошибка компиляции. Это сделано из соображений безопасности.
Зачем использовать указатели?
изменение объекта без копирования;
экономия памяти;
работа с большими структурами.
Указатель в Go – это переменная, которая хранит адрес другой переменной в памяти. Для получения адреса используется оператор &, а для получения значения по адресу – оператор * (разыменование). Указатели позволяют изменять переменные внутри функций без создания копий и эффективно работать с большими структурами данных. Нулевым значением указателя является nil. В отличие от C и C++, в Go отсутствует арифметика указателей, что повышает безопасность программ. Указатели широко используются при работе со структурами и методами для изменения исходных объектов и уменьшения затрат памяти.

7. GO: массивы, многомерные массивы, работа с массивами.
Массив (Array) – это структура данных, предназначенная для хранения фиксированного количества элементов одного типа.
_Основные свойства массива
1) Все элементы имеют одинаковый тип
Корректно: var nums [3]int
2) Размер фиксирован
После создания изменить размер массива нельзя.
var arr [5]int
Число элементов всегда будет равно 5.
3) Элементы располагаются последовательно в памяти
4) Индексация начинается с нуля
_Объявление массива
Общий синтаксис: var имя [размер]тип
Пример: var numbers [5]int
Что означает:
numbers — имя массива
5       — количество элементов
int     — тип элементов
_Значения по умолчанию
Если массив не инициализирован: var numbers [5]int
получим: [0 0 0 0 0]
Потому что для int zero value = 0.
_Инициализация массива
Полная инициализация
var numbers [5]int = [5]int{10, 20, 30, 40, 50}
Сокращенная запись:
numbers := [5]int{10, 20, 30, 40, 50}
_Автоматическое определение размера
Go может сам определить размер массива.
numbers := [...]int{10, 20, 30, 40, 50}
Компилятор автоматически вычислит: [5]int
_Размер массива
Встроенная функция: len()
_Перебор массива
Самый простой способ – цикл for.
По индексам
numbers := [5]int{10,20,30,40,50}
for i := 0; i < len(numbers); i++ {
    fmt.Println(numbers[i])
}
Цикл range
Специальный механизм Go для перебора коллекций.
for index, value := range numbers {
    fmt.Println(index, value)
}
_Массивы как тип данных
Очень важная особенность Go. Размер массива является частью его типа.
var a [3]int
var b [5]int
Это разные типы.
Нельзя: a = b. Ошибка компиляции.
_Передача массива в функцию
По умолчанию массив передается по значению, то есть его копия.
_Сравнение массивов
Массивы можно сравнивать оператором ==.
Должен быть:
одинаковый размер;
одинаковые значения элементов.
_Многомерные массивы
Массив может содержать другие массивы.
Инициализация двумерного массива
matrix := [3][3]int{
    {1,2,3},
    {4,5,6},
    {7,8,9},
}
Массив в Go – это структура данных фиксированного размера, предназначенная для хранения элементов одного типа. Размер массива является частью его типа, поэтому массивы разной длины считаются разными типами. Элементы массива индексируются начиная с нуля. Для получения размера используется функция len(). Массивы могут быть одномерными и многомерными. Наиболее распространенным видом многомерного массива является двумерный массив, который представляет собой таблицу строк и столбцов. По умолчанию массивы передаются в функции по значению, то есть копируются целиком. Для перебора элементов используются циклы for и конструкция range.

8. GO: срезы (slice), работа со срезами.
Главный недостаток массива:

var arr [5]int
Размер массива фиксирован.
Но в реальных программах количество данных обычно заранее неизвестно. Для таких задач используются срезы.
Slice (срез) – это динамическое представление массива.
Срез – это структура данных, которая предоставляет доступ к последовательности элементов массива и позволяет работать с ней как с динамической коллекцией.
Массив: [5]int
Срез: []int
_Объявление среза
1) Через литерал
nums := []int{10, 20, 30}
2) Через var
var nums []int
Создается пустой срез.
3) Создание через make()
Очень распространенный способ.
make([]тип, длина)
nums := make([]int, 5)
_Длина среза (len)
Функция: len()
_Вместимость среза (cap)
Функция: cap()
Capacity (вместимость) показывает: сколько элементов можно хранить в выделенной области памяти до перевыделения массива.
Например:
nums := make([]int, 5, 10)
Получаем:
len = 5
cap = 10
Это означает:
5 элементов используются
10 элементов выделено в памяти
_Внутреннее устройство slice
Срез НЕ хранит данные самостоятельно.
Он содержит:
Указатель на массив
Длину
Вместимость
_Добавление элементов
Для этого используется встроенная функция: append()
nums := []int{1,2,3}
nums = append(nums, 4)
Несколько элементов:
nums = append(nums, 5, 6, 7)
_Что делает append внутри?
Если места хватает: используется текущий массив
Если места не хватает: создается новый массив, копируются старые данные, добавляются новые элементы
Это одна из важнейших особенностей slice.
_Создание среза из массива
Есть массив: arr := [5]int{10,20,30,40,50}
Создаем срез: s := arr[1:4]
Очень важная особенность
Срез и массив используют одну память.
Если изменить элемент среза, который создан на основе массива, то тогда изменится и сам массив. Срез ссылается на тот же массив, копии не создаются.
_Копирование срезов
1) Не копирование данных. Копирование заголовка slice. При изменении изменяются оба.
a := []int{1,2,3}
b := a
2) Настоящее копирование с помощью функции copy(). Теперь срезы независимы.
a := []int{1,2,3}
b := make([]int, len(a))
copy(b, a)
_Удаление элемента
В Go нет встроенной функции Delete. Обычно делают так:
nums := []int{10,20,30,40}
nums = append(nums[:1], nums[2:]...)
_Перебор среза
1) Обычный цикл
for i := 0; i < len(nums); i++ {
    fmt.Println(nums[i])
}
2) Через range
for index, value := range nums {
    fmt.Println(index, value)
}
_Nil-срез
var nums []int
Состояние: nums == nil
Результат: true
_Пустой срез
Пример: nums := []int{}
nums != nil
Срез (slice) – это динамическая структура данных в Go, предназначенная для работы с последовательностями элементов. В отличие от массива, размер среза может изменяться во время выполнения программы. Срез содержит указатель на базовый массив, длину (len) и вместимость (cap). Для создания срезов используются литералы, функция make() и операции получения подмассивов. Для добавления элементов применяется функция append(), для копирования – copy(). Срезы являются ссылочным типом и обычно используются вместо массивов в большинстве Go-приложений.

9. GO: ассоциативные массивы (map), работа с ассоциативными массивами.
Map (ассоциативный массив, словарь, хеш-таблица) – это структура данных, которая хранит пары: ключ → значение
_Внутреннее устройство map
Map в Go реализован как хеш-таблица (hash table).
Ключ

 ▼

 │

 ▼
Значение
1) вычисляет хеш ключа "Анна";
2) определяет корзину;
3) сохраняет там значение.
Поэтому поиск обычно выполняется очень быстро.
_Объявление map
1) Общий синтаксис
map[типКлюча]типЗначения

Особенность – после такого объявления map еще не готов к использованию. При попытке добавить значение получим panic, потому что map имеет значение nil.
2) Создание map через make

После такого можно добавлять элементы.

users := map[string]int{

Очень просто: users["Мария"] = 28
_Получение значения

age := users["Петр"]
Если ключа нет: 0
То есть будет возвращено ZERO VALUE для того типа, который указан для значений в map.

Используется специальная форма доступа.
value, ok := map[key]
age, ok := users["Иван"]
Если ключ найден:
age = 25
ok = true
Если ключ отсутствует:
age = 0
ok = false
_Изменение значения
users["Анна"] = 30
_Удаление элемента
Используется функция: delete()
delete(map, key)
delete(users, "Иван")
Если ключа нет: delete(users, "Неизвестный")
Ошибки не будет.
_Размер map
Используется функция: len()
_Перебор map
Используется: range
for key, value := range users {
    fmt.Println(key, value)
}
Очень важная особенность – порядок обхода map НЕ гарантируется.
_Какие типы могут быть ключами?
Ключ должен быть сравнимым.
_Какие типы нельзя использовать?
Нельзя использовать типы, которые нельзя сравнивать.
map[[]int]string – Ошибка.
Срезы и map несравнимы.
_Map как ссылочный тип
Очень важная особенность.
При приравнивании map они будут указывать на одну и ту же map. Тогда при изменении одной будет меняться и другая. Потому что копируются не данные, а копируется ссылка на внутреннюю структуру map.
_Передача map в функцию
Map передается как ссылочный тип. Исходный объект изменится.
_Вложенные map
Можно хранить map внутри map.
students := map[string]map[string]int{
    "Анна": {
        "Math": 10,
        "Go": 9,
    },
}
Map в Go – это ассоциативный массив, реализованный на основе хеш-таблицы и предназначенный для хранения пар «ключ-значение». Объявляется с помощью конструкции map[типКлюча]типЗначения. Для создания обычно используется функция make(). Элементы добавляются и изменяются через обращение по ключу, удаляются функцией delete(). Для проверки существования ключа используется форма value, ok := map[key]. Размер map определяется функцией len(), а перебор выполняется через range. Map является ссылочным типом и не гарантирует сохранение порядка элементов.

Структуры – одна из самых важных тем в Go. Если в объектно-ориентированных языках основным инструментом являются классы, то в Go роль классов во многом выполняют именно структуры.
Структура (struct) – это пользовательский составной тип данных, который объединяет несколько связанных полей под одним именем.
Структура – это пользовательский тип данных, состоящий из набора полей, которые могут иметь различные типы.
Благодаря структурам появляется возможность удобно хранить связанные данные в одном месте.
_Объявление структуры
type ИмяСтруктуры struct {
    Поле Тип

user := User{

читаемо;
порядок полей не важен;

3) Через var

Получим значения по умолчанию:
Name = ""
Age  = 0
_Zero Value структуры
Если структура создана без инициализации:
var user User
Каждое поле получает значение по умолчанию.
_Сравнение структур
Структуры можно сравнивать оператором: == если все их поля сравнимы.
_Передача структуры в функцию
По умолчанию структура передается по значению. То есть исходный объект не изменится, потому что создастся копия структуры.
Если нужно, чтобы можно было изменять, то нужно передавать ссылку и принимать по указателю.
_Встраивание структур (Embedding)
Особенность Go.
type Person struct {
    Name string
}
type Employee struct {
    Person
    Salary int
}
Создание:
emp := Employee{
    Person: Person{
        Name: "Анна",
    },

}
Благодаря этому мы можем напрямую обращаться emp.Name, хотя поле находится внутри Person. Это механизм, похожий на наследование, но не является наследованием в классическом понимании.
_Структуры и JSON
Очень часто используются в веб-разработке.
type User struct {
    Name string `json:"name"`

Go автоматически сопоставляет поля структуры с полями JSON.
Структура (struct) в Go – это пользовательский составной тип данных, объединяющий набор связанных полей различных типов. Структуры используются для моделирования объектов предметной области и являются основным средством организации данных в Go. Поля структуры доступны через оператор точки. Структуры могут содержать другие структуры, передаваться в функции по значению или по указателю, а также иметь собственные методы. Вместе с методами и интерфейсами структуры образуют основу построения приложений в Go и частично заменяют классический механизм классов объектно-ориентированных языков.

13.GO: строки, работа со строками
Строка в Go имеет тип: string
Строка (string) – это неизменяемая последовательность байтов, содержащая текстовые данные в кодировке UTF-8.
_Объявление строки
1) Через var
var name string = "Диана"
2) С выводом типа
var name = "Диана"
3) Через краткую запись
name := "Диана"
_Строковые литералы
1) Обычные строки. 
Используются двойные кавычки.
name := "Hello"
2) Raw-строки
Используются обратные кавычки.
text := `Hello
World`
 
_Внутреннее устройство строки
Строка в Go хранится как последовательность байтов. После создания строку изменить нельзя, будет ошибка компиляции при попытке.
_Длина строки
Используется функция: len()
Важно то, что эта функция возвращает не количество символов, а количество байт. В UTF-8 русская буква обычно занимает 2 байта.
_Доступ к символам строки
Строка индексируется как массив.
s := "Hello"
fmt.Println(s[0])
Результат: 72
Почему не H? Потому что возвращается байт.
Получить символ: fmt.Printf("%c", s[0])
_Rune и Unicode
На самом деле: rune = int32
Rune хранит Unicode-символ.
Почему нужен rune? Потому что символы Unicode могут занимать несколько байтов.
_Перебор строки
1) Через индексы
s := "Hello"

for i := 0; i < len(s); i++ {
    fmt.Println(s[i])
}
Получаем байты.
2) Через range
Наиболее правильный способ.
s := "Привет"
for _, r := range s {
    fmt.Printf("%c\n", r)
}
Получаем символы. Range автоматически работает с Unicode.
_Преобразование строки в срез rune
Иногда нужно работать именно с символами.
s := "Привет"
runes := []rune(s)
Теперь: fmt.Println(len(runes))
Результат: 6
Количество символов, а не байтов.
_Конкатенация строк
Оператор: +
first := "Hello"
second := "World"
result := first + " " + second
_Сравнение строк
Используются обычные операторы.
a := "Hello"
b := "Hello"
fmt.Println(a == b)
Также можно: <, >, <=, >=
Сравнение выполняется лексикографически.
_Пакет strings
Для работы со строками используется стандартный пакет:
import "strings"
1) Contains
Проверяет наличие подстроки. Возвращает bool.

2) HasPrefix
Проверяет начало строки. Возвращает bool.

3) HasSuffix
Проверяет окончание строки. Возвращает bool.
strings.HasSuffix("file.txt", ".txt")
4) ToUpper

Переводит в верхний регистр.
strings.ToUpper("hello")
5) ToLower
Переводит в нижний регистр.
strings.ToLower("HELLO")
6) Replace
Замена подстроки.
strings.ReplaceAll(
    "Go is good",
    "good",
    "awesome",
)
7) Split
Разделение строки.
strings.Split("a,b,c", ",")
Результат: []string{"a","b","c"}	
8) Join
Объединение строк.
strings.Join(
    []string{"a","b","c"},
    "-"
)
Результат: a-b-c
9) TrimSpace
Удаление пробелов.
strings.TrimSpace("   hello   ")
10) strings.Index()
Поиск подстроки. Если не найдет, то вернет -1.
strings.Index("Hello", "ll")
_Форматирование строк
Используется пакет: fmt
_Строки и []byte
Строку можно преобразовать:
s := "Hello"
data := []byte(s)
Получаем: [72 101 108 108 111]
Обратно: s2 := string(data)
_Strings Builder
Если нужно собрать большую строку.
var builder strings.Builder
builder.WriteString("Go")
result := builder.String()
Строка (string) в Go представляет собой неизменяемую последовательность байтов в кодировке UTF-8. Для получения длины используется функция len(), которая возвращает количество байтов. Доступ к элементам осуществляется по индексу, однако для корректной работы с Unicode-символами используются тип rune и цикл range. Основные операции со строками включают конкатенацию, сравнение, поиск, разделение и объединение строк. Для работы со строками применяется стандартный пакет strings, а для эффективного формирования больших строк используется тип strings.Builder. Строки можно преобразовывать в []byte и обратно.

14. GO: операции (присваивание, бинарные, унарные, логические).  
_Что такое оператор и операция?
Оператор – символ, обозначающий действие.
Например: 
+
-
*
/
=
==
&&
Операция – само действие над операндами.
Например: 10 + 5
Операнды: 10 5
Операция: сложение
_Классификация операций в Go
Основные группы:
1. Присваивание
2. Арифметические

Используются для записи значения в переменную.

x := 10

var x int

Особенность Go.
a, b := 10, 20
Получаем:
a = 10

В Go есть встроенный механизм обмена.

a := 10

Временная переменная не нужна.
4) Присваивание с действием
+=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=

3) Умножение
a * b
4) Деление
a / b

2) Логическое ИЛИ

3) Логическое НЕ

Короткое замыкание (Short Circuit)

6) ||
7) Присваивание
В Go операции делятся на операции присваивания, арифметические (бинарные), операции сравнения, логические, унарные и побитовые. Операции присваивания используются для записи значений в переменные. Арифметические операции включают сложение, вычитание, умножение, деление и получение остатка от деления. Операции сравнения возвращают логическое значение true или false. Логические операции &&, || и ! используются для работы с булевыми выражениями. Унарные операции работают с одним операндом и включают отрицание числа, логическое отрицание, получение адреса переменной и разыменование указателя. Также Go поддерживает побитовые операции и операции сдвига. Все операции выполняются в соответствии с установленным приоритетом.

15. GO: конструкции языка: if, switch
_Конструкция if
Оператор if позволяет выполнить блок кода только при выполнении некоторого условия.
if условие {
    код
}
Условие должно быть bool
if age {
}
Тогда будет ошибка компиляции.
В Go условие всегда должно иметь тип bool
_Блок else
Позволяет выполнить альтернативный код.
if условие {
    код1
} else {
    код2
}
_Цепочка else if
Используется при нескольких вариантах.
if условие1 {
   код1
} else if условие2 {
   код2
} else if условие3 {
   код3
} else {
   код4
}
_Короткое объявление внутри if
Особенность Go.
Можно создать переменную прямо внутри условия.
if x := 10; x > 5 {
    fmt.Println(x)
}
Но область видимости переменной ограничена блоком if.
_Очень популярный пример с error
Практически в каждом Go-проекте встречается:
result, err := divide(10, 2)
if err != nil {
    fmt.Println(err)
}
_Конструкция switch
Используется вместо длинной цепочки:
if
else if
else if
else if
…
_Назначение switch
Позволяет выбрать один вариант из нескольких.
switch выражение {
case значение1:
    код
case значение2:
    код
default:
    код
}
Проверка выполняется сверху вниз. При совпадении выполняется соответствующий блок. 
В Go после выполнения case происходит автоматический выход.
Никакой break писать не нужно. Он выполняется автоматически.
_Несколько значений в case
switch day {
case 1, 2, 3, 4, 5:
    fmt.Println("Рабочий день")
case 6, 7:
    fmt.Println("Выходной")
}
_Switch без выражения
В Go можно написать:
switch {
case age < 18:
    fmt.Println("Ребенок")
case age < 65:
    fmt.Println("Взрослый")
default:
    fmt.Println("Пенсионер")
}
Фактически это аналог:
if
else if
else if
_Fallthrough
Особая конструкция Go. Позволяет перейти к следующему case. Короче типо как в с/с++, если не поставить break.
switch 1 {
case 1:
    fmt.Println("One")
    fallthrough
case 2:
    fmt.Println("Two")
}
Результат:
One
Two
Без fallthrough:
One
_Инициализация внутри switch
Как и в if.
switch x := 10; {
case x > 5:
    fmt.Println("Большое число")
}
Переменная существует только внутри switch.
_Type Switch
Используется при работе с интерфейсами.
var value interface{} = "Hello"
switch v := value.(type) {
case string:
    fmt.Println("Строка", v)
case int:
    fmt.Println("Число", v)
}
Результат: Строка Hello
На базовом уровне достаточно знать, что Type Switch позволяет определить реальный тип значения интерфейса.
_Сравнение if и switch
If лучше использовать когда:
сложные условия, >, <, >=, <=, &&, ||
switch лучше использовать когда: много вариантов выбора
Конструкции if и switch используются для организации ветвления программы. Оператор if выполняет блок кода при истинности логического условия и может содержать ветви else и else if. В Go условие обязательно должно иметь тип bool. Конструкция switch позволяет выбирать один из нескольких вариантов выполнения программы на основе значения выражения или логических условий. В отличие от языка C, в Go после выполнения case автоматически происходит выход из конструкции, поэтому оператор break обычно не требуется. Также поддерживаются default, fallthrough и специальный вариант type switch для работы с интерфейсами.

16. GO: конструкции языка: циклы
Цикл – это конструкция языка, которая позволяет многократно выполнять один и тот же блок кода до выполнения некоторого условия.
¬¬_Особенность Go
В большинстве языков есть несколько видов циклов:
for
while
do while
foreach
В Go ситуация другая. В языке существует только одна циклическая конструкция: for
Но она может использоваться как:
for
while
foreach
бесконечный цикл
Поэтому часто говорят: В Go существует только один цикл – for.
_Классический цикл for
Самый распространенный вариант.
for инициализация; условие; изменение {
    код
}
for i := 0; i < 5; i++ {
    fmt.Println(i)
}
Переменная цикла существует только внутри цикла.
_Цикл как while
В Go нет отдельного цикла while. Его роль выполняет:
for условие {
}
x := 0
for x < 5 {
    fmt.Println(x)
    x++
}
_Бесконечный цикл
Создается так:
for {
}
for {
    fmt.Println("Working...")
}
Часто используется:
серверами;
обработчиками сообщений;
системными службами.
_Выход из цикла – break
break полностью завершает цикл.
for i := 0; i < 10; i++ {
    if i == 5 {
        break
    }
    fmt.Println(i)
}
_Пропуск итерации – continue
continue прерывает текущую итерацию и переходит к следующей.
for i := 0; i < 5; i++ {
    if i == 2 {
        continue
    }
    fmt.Println(i)
}
_Вложенные циклы
Цикл может находиться внутри другого цикла.
for i := 1; i <= 3; i++ {
    for j := 1; j <= 3; j++ {
        fmt.Println(i, j)
    }
}
_Цикл range
Очень важная конструкция Go. Используется для перебора коллекций.
Может работать с:
массивами
срезами
строками
map
каналами
Range для массива
numbers := [3]int{10, 20, 30}
for index, value := range numbers {
    fmt.Println(index, value)
}
Что возвращает range?
Всегда два значения:
индекс
значение
Если индекс не нужен, то используется специальный идентификатор: 
for _, value := range numbers {
    fmt.Println(value)
}
_Метки (Labels)
Используются редко, но иногда встречаются.
Outer:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if i == 1 && j == 1 {
            break Outer
        }
    }
}
Здесь break Outer завершает сразу оба цикла. Потому что у нас внешний цикл помечен этой меткой.
В Go существует единственная циклическая конструкция – for, которая может использоваться как классический цикл, как аналог while и как бесконечный цикл. Для досрочного завершения цикла используется оператор break, а для пропуска текущей итерации – continue. Для перебора коллекций применяется конструкция range, которая поддерживает массивы, срезы, строки, ассоциативные массивы и каналы. При работе со строками range корректно обрабатывает Unicode-символы. Циклы могут быть вложенными, а также могут использовать метки для управления выходом из нескольких уровней вложенности.

17. GO: функции, применение функций.
Функция (function) – это именованный блок кода, который выполняет определенную задачу и может быть вызван из различных частей программы.
Функция – это самостоятельный программный модуль, содержащий набор инструкций, который может принимать параметры и возвращать результат.

execute(func() {
    fmt.Println("Run")
})
_Variadic-функции
Функции с переменным количеством параметров.
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}
_Передача аргументов
По умолчанию Go передает аргументы по значению. Но можно передавать и через указатель.
_Область видимости
Переменные функции существуют только внутри функции.
Функция в Go представляет собой именованный блок кода, предназначенный для выполнения определенной задачи. Функции объявляются с помощью ключевого слова func, могут принимать параметры и возвращать одно или несколько значений. В Go широко используется механизм множественного возврата, особенно для передачи результата и ошибки. Функции могут быть анонимными, передаваться как параметры, возвращаться из других функций и поддерживать переменное количество аргументов через механизм variadic-параметров. По умолчанию аргументы передаются по значению, однако для изменения исходных данных используются указатели. Функции являются основным механизмом организации и повторного использования кода в приложениях Go.
18. GO:  горутины, применение горутин.
Фактически именно благодаря горутинам Go стал настолько популярным для:
серверов
микросервисов
сетевого программирования
облачных приложений
систем обработки данных
высоконагруженных систем
Если структуры и функции есть почти в любом языке, то горутины – одна из ключевых особенностей именно Go.
_Что такое конкурентность?
Конкурентность (Concurrency) – выполнение нескольких задач независимо друг от друга.
Задача 1 → загрузка файла
Задача 2 → обработка запроса
Задача 3 → запись лога
Все задачи могут выполняться одновременно или почти одновременно.
_Что такое горутина?
Горутина (goroutine) – это легковесный поток выполнения, управляемый средой выполнения Go (Go Runtime).
Горутина – это независимая функция или метод, выполняемый конкурентно с другими горутинами внутри процесса Go.
Обычный вызов:
func work() {
    fmt.Println("Work")
}
work()
означает:
Выполнить функцию здесь и сейчас
Дождаться окончания
Продолжить выполнение
Горутина:
go work()
означает:
Запусти функцию отдельно
Не жди ее завершения
Продолжай выполнение программы
_Создание горутины
Ключевое слово: go
Это единственное, что отличает обычный вызов от запуска горутины.
_Первый пример
package main
import (
    "fmt"
    "time"
)
func hello() {
    fmt.Println("Hello")
}
func main() {
    go hello()
    time.Sleep(time.Second)
}
_Почему нужен Sleep?
Программа может завершиться раньше, чем горутина успеет выполниться без sleep.
_Как работают горутины внутри?
Горутина ≠ поток операционной системы.
В ОС существуют потоки: Thread
Они достаточно тяжелые.
Создание потока требует ресурсов.
Горутины намного легче.
_Планировщик Go
За горутинами следит специальный компонент: Go Scheduler
Он решает:
какая горутина выполняется
когда переключиться
на каком потоке ОС выполнять
Поэтому важное правило:
Никогда нельзя рассчитывать на порядок выполнения горутин.
_Анонимные горутины
Очень распространены.
go func() {
    fmt.Println("Hello")
}()
Что происходит?
Создается анонимная функция.
Она запускается как горутина.
_Горутины и функции
Можно передавать параметры.
go func(name string) {
    fmt.Println(name)
}("Диана")
_WaitGroup
Для ожидания завершения горутин используется:
sync.WaitGroup
Это наиболее правильный способ.
package main
import (
    "fmt"
    "sync"
)
func main() {
    var wg sync.WaitGroup
    wg.Add(1)
    go func() {
       defer wg.Done()
        fmt.Println("Hello")
    }()
    wg.Wait()
}
_Для чего применяются горутины?
1) Веб-серверы
Каждый запрос можно обрабатывать отдельно.
Клиент 1
Клиент 2
Клиент 3
↓
Отдельные горутины
2) Загрузка файлов
Параллельная загрузка нескольких файлов.
3) Микросервисы
Одновременная обработка запросов.
4) Работа с сетью
Параллельное обслуживание подключений.
5) Фоновые задачи
логирование
очистка кэша
мониторинг
_Горутины и общая память
Очень важный момент. Если две горутины работают с одной переменной: counter++ может возникнуть Race Condition. То есть состояние гонки. Для решения используются: Mutex, Channels, Atomic операции
Горутина (goroutine) – это легковесный поток выполнения, управляемый средой выполнения Go. Горутины создаются с помощью ключевого слова go перед вызовом функции и позволяют выполнять несколько задач конкурентно. В отличие от потоков операционной системы, горутины занимают мало памяти и эффективно планируются встроенным планировщиком Go Runtime. Для ожидания завершения горутин обычно используется sync.WaitGroup. Горутины широко применяются в веб-серверах, микросервисах, сетевых приложениях, фоновых задачах и системах параллельной обработки данных. При совместной работе с общей памятью необходимо использовать механизмы синхронизации для предотвращения состояний гонки (race condition).

20. GO: применение пакета database/sql для работы с реляционной базой данных.
Пакет: import "database/sql"
предоставляет единый интерфейс для работы с различными СУБД.
Очень важно понимать: 
database/sql НЕ умеет работать с БД самостоятельно. Он предоставляет только API. Для конкретной СУБД нужен драйвер.
_Архитектура работы
Приложение Go
       │
       ▼
database/sql
       │
       ▼
Драйвер БД
       │
       ▼
СУБД
_Что такое драйвер?
Драйвер – это библиотека, которая знает, как общаться с конкретной СУБД.
Например:
PostgreSQL понимает свой протокол
MySQL понимает свой протокол
SQL Server понимает свой протокол
Драйвер переводит вызовы Go в команды конкретной БД.
_Подключение к БД
Основная функция: sql.Open()
Возвращает: *sql.DB
Очень важный момент: sql.Open()
не открывает соединение немедленно. Она лишь создает объект подключения.
_Проверка подключения
Для реальной проверки используется: db.Ping()
err = db.Ping()
if err != nil {
    log.Fatal(err)
}
_Закрытие соединения
После завершения работы: defer db.Close()
_Объект sql.DB
Многие думают: sql.DB = одно соединение
Это неверно.
На самом деле: sql.DB = пул соединений
Поэтому db обычно создается один раз на все приложение.
_Выполнение команд без результата
Используется: db.Exec()
Подходит для:
INSERT
UPDATE
DELETE
CREATE TABLE
DROP TABLE
_, err := db.Exec(
    "INSERT INTO users(name) VALUES($1)",
    "Диана",
)
_Получение одного значения
Используется: QueryRow()
row := db.QueryRow(
    "SELECT name FROM users WHERE id = $1",
    1,
)
Результат нужно считать через: Scan()
var name string
err := row.Scan(&name)
Что делает Scan?
Она копирует данные из результата SQL-запроса в переменные Go.
_Получение нескольких строк
Используется: db.Query()
rows, err := db.Query(
    "SELECT id, name FROM users",
)
Получаем объект: *sql.Rows
_Перебор результатов
Используется цикл: for rows.Next()
for rows.Next() {
    var id int
    var name string
    rows.Scan(&id, &name)
    fmt.Println(id, name)
}
Next() перемещает курсор к следующей записи.
_Закрытие Rows
После работы: defer rows.Close()
_Структуры и Scan
Очень популярный подход.
Создаем структуру:
type User struct {
    ID   int
    Name string
}
Читаем данные:
var users []User
for rows.Next() {
    var u User
    rows.Scan(
        &u.ID,
        &u.Name,
    )
    users = append(users, u)
}
_Подготовленные запросы
Используется: Prepare()
stmt, err := db.Prepare(
    "INSERT INTO users(name) VALUES($1)",
)
После этого можно так:
stmt.Exec("Анна")
stmt.Exec("Иван")
stmt.Exec("Олег")
_Транзакции
Транзакция – группа операций, которая должна выполниться полностью или не выполниться вообще.
1) Начало транзакции
tx, err := db.Begin()
Получаем: *sql.Tx
2) Выполнение запросов
tx.Exec(...)
3) Подтверждение
tx.Commit()
Сохранить изменения.
4) Откат
tx.Rollback()
Отменить изменения.
_Работа с NULL
Предположим: name = NULL
Нельзя писать: var name string
Используются специальные типы.
Например: sql.NullString
Проверка:
if name.Valid {
    fmt.Println(name.String)
}
Также существуют:
sql.NullInt64
sql.NullBool
sql.NullFloat64
sql.NullTime
_Настройка пула соединений
Можно управлять количеством соединений.
1) Максимум открытых
db.SetMaxOpenConns(25)
2) Максимум простаивающих
db.SetMaxIdleConns(10)
3) Время жизни соединения
db.SetConnMaxLifetime(...)
Для высоконагруженных приложений это очень важно.
Пакет database/sql предоставляет стандартный интерфейс для работы с реляционными базами данных в Go. Для взаимодействия с конкретной СУБД требуется соответствующий драйвер. Основным объектом является sql.DB, представляющий пул соединений. Для выполнения запросов используются методы Exec(), Query() и QueryRow(), а для чтения данных – метод Scan(). Пакет поддерживает подготовленные запросы (Prepare()), транзакции (Begin(), Commit(), Rollback()), работу с NULL-значениями и настройку пула соединений. database/sql является основным механизмом доступа к данным в приложениях Go, работающих с реляционными базами данных.

21. GO: применение фреймворка GORM
ORM (Object Relational Mapping) – технология отображения объектов программы на таблицы реляционной базы данных.
GORM – самая популярная ORM-библиотека для Go.
GORM позволяет:

создавать таблицы
добавлять записи
обновлять данные
удалять записи
выполнять запросы
работать со связями
использовать транзакции
автоматически мигрировать БД
_Почему используют GORM?
Без GORM:
db.Query(
    "SELECT id, name FROM users WHERE id = $1",
    id,
)
С GORM:
db.First(&user, id)
Код получается значительно короче и понятнее.
_Установка GORM
1) Основной пакет:
go get gorm.io/gorm
2) Для PostgreSQL:
go get gorm.io/driver/postgres
3) Для MySQL:
go get gorm.io/driver/mysql
4) Для SQL Server:
go get gorm.io/driver/sqlserver
5) Для SQLite:
go get gorm.io/driver/sqlite
_Подключение к БД
Пример PostgreSQL:
import (
    "gorm.io/driver/postgres"
    "gorm.io/gorm"
)
Подключение:
dsn := "host=localhost user=postgres password=123 dbname=test port=5432 sslmode=disable"
db, err := gorm.Open(
    postgres.Open(dsn),
    &gorm.Config{},
)
Получаем объект: *gorm.DB
Это основной объект для работы с GORM.
_Создание модели
В GORM таблица обычно описывается структурой.
type User struct {
    ID   uint
    Name string
    Age  int
}
Что произойдет?
GORM автоматически создаст таблицу: users с
id
name
age
_Встроенная модель gorm.Model
Очень часто используется.
type User struct {
    gorm.Model
    Name string
    Age  int
}
Внутри gorm.Model уже есть поля:
ID        uint
CreatedAt time.Time
UpdatedAt time.Time
DeletedAt gorm.DeletedAt
То есть таблица автоматически получает:
id
created_at
updated_at
deleted_at
_Автоматическая миграция
Одна из самых популярных возможностей GORM.
Используется: db.AutoMigrate(&User{})
Что делает?
создает таблицу
создает столбцы
обновляет структуру таблицы
Если таблица уже существует: данные не удаляются
_Создание записи (INSERT)
Используется: db.Create()
user := User{
    Name: "Диана",
    Age: 20,}
db.Create(&user)
_Получение одной записи
Используется: db.First()
var user User
db.First(&user, 1)
Получаем запись: ID = 1
SQL аналог: SELECT * FROM users WHERE id = 1 LIMIT 1;
_Получение нескольких записей
Используется: db.Find()
var users []User
db.Find(&users)
SQL аналог: SELECT * FROM users;
_Условный поиск
Используется: Where()
db.Where(
    "age > ?",
    18,
).Find(&users)
SQL аналог: SELECT * FROM users WHERE age > 18;
_Получение первой подходящей записи
db.Where(
    "name = ?",
    "Диана",
).First(&user)
_Обновление записи
Используется: Update()
db.Model(&user).Update("Age", 25)
SQL аналог: UPDATE users SET age = 25 WHERE id = ...
_Обновление нескольких полей
db.Model(&user).
Updates(User{
    Name: "Анна",
    Age: 30,
})
_Удаление записи
Используется: Delete()
db.Delete(&user)
SQL аналог: DELETE FROM users WHERE id = ...
_Soft Delete
Если структура содержит: gorm.Model, то удаление обычно становится логическим.
Что происходит?
Вместо: DELETE
выполняется: UPDATE
Заполняется поле: deleted_at
То есть запись остается в БД.
_Поиск по ID
db.First(&user, 10)
SQL: SELECT * FROM users WHERE id = 10 LIMIT 1;

_Сортировка
Используется: Order()
db.Order("age desc").Find(&users)
SQL: ORDER BY age DESC
_Ограничение количества строк
Используется: Limit()
db.Limit(5).Find(&users)
SQL: LIMIT 5
_Смещение (Pagination)
Используется: Offset()
db.Limit(10).
   Offset(20).
   Find(&users)
SQL: LIMIT 10 OFFSET 20

GORM – это популярный ORM-фреймворк для Go, предназначенный для упрощения работы с реляционными базами данных. Он позволяет отображать структуры Go на таблицы БД и выполнять операции CRUD без написания большого количества SQL-кода. Основным объектом является *gorm.DB. GORM поддерживает автоматические миграции через AutoMigrate(), создание записей (Create()), поиск (First(), Find()), фильтрацию (Where()), обновление (Update(), Updates()), удаление (Delete()), транзакции, связи между таблицами и механизм логического удаления (Soft Delete). Благодаря этому GORM значительно ускоряет разработку приложений, работающих с реляционными базами данных.
